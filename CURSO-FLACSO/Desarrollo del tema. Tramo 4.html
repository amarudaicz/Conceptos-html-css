Transiciones en CSS 










La propiedad transition de CSS posibilita que una o más declaraciones pasen de un estado inicial a uno final de manera gradual. El método abreviado (shorthand) tiene la siguiente sintaxis: 

transition: propiedad duración función-de-temporizador-(opcional) demora-(opcional);

Como ven, primero definimos la propiedad a interpolar, luego establecemos el tiempo en segundos que tomará el pasaje de un estado al otro, a continuación el nombre de una función que determinará la manera en que se realizará la transición y por último, el tiempo de retardo tras el cual comenzará el proceso. Cada uno de esos ítems estará separado por un espacio y será escrito de manera obligatoria en el orden señalado.

Veamos un ejemplo:

<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Transiciones</title> 
<style>
*{
    margin:0;
    padding:0;
}
body{
    background:#CACACA;
}

.rojo{
    width:100px;
    height:100px;
    border-radius:50%;
    margin:50px;
    background:red;
    transition:background 2s;
    cursor:pointer;
}
.rojo:hover{
    background:black;
}
</style>
</head> 
<body> 
    <div class="rojo"></div>
</body> 
</html>

En el ejemplo hemos definido una transición para la propiedad background a cumplirse en 2 segundos. Como no determinamos una función de transición, el sistema asume el valor por defecto linear, que hace que la animación transcurra de principio a fin a la misma velocidad. Otro ítem que omitimos es el de retardo o demora, que cuando no se especifica adquiere el valor cero y entonces la animación comienza de manera inmediata. Solo esos 2 últimos valores (el correspondiente al nombre de la función de transición y el del retardo o demora) son opcionales en la sintaxis abreviada de la propiedad transition.


Las funciones de transición, que como mencionamos determinan la velocidad de la interpolación, son las siguientes:


linear

El efecto de transición pasa con la misma velocidad de principio a fin.

ease

La transición es lenta al inicio, después rápida, después lenta de nuevo.

ease-in

La transición es lenta al inicio.

ease-out

La transición es lenta al final.

ease-in-out

La transición es lenta al inicio y al final.

Pueden definirse transiciones correspondientes a varias propiedades separándolas con coma. Y si queremos referirnos a todas las propiedades en simultáneo podemos utilizar la clave all como comodín:

<!DOCTYPE html> 
<html>

<meta charset="utf-8"> 
<title>Transiciones</title> 
<style>
*{
    margin:0;
    padding:0;
}
body{
    background: hsla(0, 100%, 50%,0.5);
}

.rojo{
    width:100px;
    height:100px;
    border-radius:50%;
    margin:50px;
    background:red;
    transition:background 2s, width 1s, height 2s;
    cursor:pointer;
}
.rojo:hover{
    background:yellow;
    width:200px;
    height:200px;
}
.azul{
    width:100px;
    height:100px;
    border-radius:50%;
    margin:50px;
    background:blue;
    transition:all 1s ease-in-out;
    cursor:pointer;
}
.azul:hover{
    background:yellow;
    width:200px;
    height:200px;
}
</style>

    <div class="rojo"></div>

</body>

La propiedad opacity de CSS permite definir la transparencia de los elementos html. En ese sentido, actúa de manera semejante al cuarto argumento de las funciones rgba o hsla que usamos en algunas oportunidades para establecer colores. Propiedad CSS Overflow

La propiedad overflow de CSS permite elegir qué hacer visualmente con el contenido que no cabe en el rectángulo determinado por las propiedades width y height de un elemento de bloque: recortarlo, mostrarlo detrás de barras de desplazamiento (scroll) o mostrarlo completo aunque no quepa en el espacio definido.

Valor de la propiedad

Comportamiento

visible

Es el valor por defecto. El contenido no es recortado y, si hay excedente, este es mostrado fuera del contenedor.

hidden

El contenido excedente es recortado y no se muestran las barras de scroll.

scroll

El contenido excedente no es recortado y las barras de scroll son mostradas siempre, aunque no sean  necesarias. 

auto

El contenido excedente no es recortado y las barras de scroll solo aparecen si es necesario.

Hay varias maneras de centrar elementos html con CSS, cada una con sus pros y contras. Y cada día aparecen nuevas, como las que utilizan las cajas flexibles (flexbox), que no vemos en este curso.

En realidad, el centrado horizontal más o menos es sencillo y ya vimos que para centrar textos o elementos inline o inline-block en una caja basta con la propiedad text-align puesta a center. Y que para centrar elementos de bloque podíamos usar margin con valor auto para left y right:


<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Centrado</title> 
<style>
.container{
    width:90%;
    margin:50px auto;
    border:1px dashed #ccc;
}
.text-center{
    text-align:center;
}
.a{
    background:orange;
}
.b{
    background:orange;
    display:inline-block;
}
.c{
    background:red;
    width:50%;
    margin:50px auto;
}
</style>
</head> 
<body> 
    <div class="container">
        <div class="text-center">
            este es un texto
        </div>
        <div class="text-center">
            <span class="a">este es un elemento inline</span> <span class="b">este es un elemento inline-block</span>
        </div>
        <div class="c">este es un elemento de bloque</div>
    </div>
</body> 
</html>

El inconveniente se da en el centrado vertical. Para textos de una sola línea vimos la posibilidad de usar  line-height o padding en aquellos casos en que la altura de la caja contenedora no estaba definida:

Noten cómo un line-height igual a la altura del contenedor falla con textos multilínea y cómo padding solo funciona bien con cajas con height auto.

Una de las opciones para centrado vertical y/o horizontal es la utilización de posicionamiento absoluto combinado con márgenes negativos. Veámos cómo hacerlo en 2 pasos. Primero, definiremos el posicionamiento absoluto de la caja:


<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Centrado</title> 
<style>
.container{
    width:90%;
    height:300px;
    margin:50px auto;
    border:1px dashed #ccc;
    background:yellow;
    position:relative;
}

.a{
    width:200px;
    height:100px;
    position:absolute;
    background:red;
    top:50%;
    left:50%;
}
</style>
</head> 
<body> 
    <div class="container">
        <div class="a">
        </div>
    </div>
</body> 
</html>

Al elemento con clase .container le establecimos un posicionamiento relativo para que sus descendientes directos con posicionamiento absoluto usen de referencia sus límites para definir las distancias de las propiedades left, top, right y bottom. En el ejemplo solo usamos las 2 primeras y vemos que la caja roja  ubica su vértice superior izquierdo en el centro exacto del contenedor amarillo. Esto, aunque esperable, no satisface nuestro deseo de centrado. Para que el rectángulo rojo quede exactamente en el centro del amarillo deberíamos poder moverlo un 50% de su ancho hacia la izquierda y un 50% de su alto hacia arriba. Y eso lo lograremos utilizando la propiedad margin. El problema es que no podremos hacerlo en porcentaje, ya que los márgenes en porcentaje se calculan con base al alto y ancho del contenedor, que en este caso es la caja amarilla. Así que deberemos escribirlos nosotros calculándolos con base a los 200px por 100px que mide nuestra caja roja:

<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Centrado</title> 
<style>
.container{
    width:90%;
    height:300px;
    margin:50px auto;
    border:1px dashed #ccc;
    background:yellow;
    position:relative;
}

.a{
    width:200px;
    height:100px;
    position:absolute;
    background:red;
    top:50%;
    left:50%;
    margin-left:-100px;/*50% del width de la caja roja, con signo negativo*/
    margin-top:-50px;/*50% del height de la caja roja, con signo negativo*/
}
</style>
</head> 
<body> 
    <div class="container">
        <div class="a">
        </div>
    </div>
</body> 
</html>

No está mal, pero tiene como problema la necesidad de conocer el ancho y/o alto de la caja a centrar. Es decir, no funcionará con cajas de dimensiones variables o definidas en medidas relativas, como por ejemplo porcentajes.

Otra manera más efectiva para realizar el centrado es mediante la utilización de la propiedad transform de CSS. Dicha propiedad permite hacer varias transformaciones 2D y 3D, como mover, girar, escalar y deformar elementos html. Su alcance total escapa al temario de este curso, por lo que les pediremos a los que estén interesados en ella que indaguen en los enlaces de la sección “Para saber  más”. Nosotros solo veremos cómo usarla para mover objetos con su función translate, que tiene la siguiente sintaxis:

transform : translate(desplazamiento-horizontal, desplazamiento-vertical);


La función translate admite 2 parámetros. El primero corresponde a la componente horizontal del desplazamiento y el segundo a la componente vertical. Son distancias que se calculan a partir de la ubicación del elemento, y que pueden definirse con unidades de longitud absoluta, como px, pt, cm, etc. o relativas, como %. Cuando son relativas se calculan con base en el ancho y alto del objeto a desplazar, lo que representa una gran ventaja con respecto a margin, ya que permite centrar objetos de dimensiones variables o cuyo ancho y/o alto desconocemos. Veamos cómo funcionaría el ejemplo anterior usando translate en vez de margin:


<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Centrado</title> 
<style>
.container{
    width:90%;
    height:300px;
    margin:50px auto;
    border:1px dashed #ccc;
    background:yellow;
    position:relative;
}

.a{
    width:200px;
    height:100px;
    position:absolute;
    background:red;
    top:50%;
    left:50%;
    transform:translate(-50%,-50%);
}
</style>
</head> 
<body> 
    <div class="container">
        <div class="a">
        </div>
    </div>
</body> 
</html>

Observen que no existe ninguna diferencia visual respecto del ejemplo en el que usamos margin. 


En el siguiente cuadro mostraremos un caso en el que las medidas de la caja roja son relativas:


<!DOCTYPE html> 
<html>

Cuando trabajamos en diseño adaptativo (responsive design), hay ocasiones en que necesitamos que la altura o el ancho o ambas medidas de una caja no crezca más allá de un valor determinado. Por ejemplo, si tenemos un cuadro informativo centrado en pantalla es posible que para desktop un ancho de 700 px sea razonable, pero el problema es que ese ancho de 700 px para un celular tipo, que mide 360 px de ancho, no cabrá en pantalla. Propiedades min-width y min-height de CSS

Así como hay casos en los cuales es necesario establecer un ancho o un alto máximos, a veces se presenta el caso contrario y nos vemos obligados a definir un espacio de trabajo mínimo. Es en esas ocasiones en que resultan útiles las propiedades min-width y min-height, que funcionan exactamente igual a max-width y max-height pero para topes de mínima en vez de máxima:


<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>max-width</title>
<style>
*{padding:0;margin:0;}
.footer,.header{
    background:#000;
    color:#fff;
    font-family:Tahoma, Geneva, sans-serif;
    text-align:center;
    line-height:100px;
    height:100px;
}
.contenido{
    font-family:Tahoma, Geneva, sans-serif;
    height:auto;
    padding:30px;
    box-sizing:border-box;
}
</style>
</head>
<body>
    <div class="header">Este es el header o cabecera</div>

Ámbitos de las variables

Ir a la página principal
Siguiente


Se denomina ámbito, alcance o scope de una variable al entorno dentro del cual la variable es operativa. Fuera de ese límite o radio de acción se considera que la variable es inexistente. Cuando una variable se declara dentro de una función, la misma tiene impacto únicamente dentro de dicha función. Se dice en ese caso que la variable es local o que tiene un alcance local a la función. Cuando una variable se declara fuera de una función, se denomina global y está disponible en todo el documento.

<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8">
<title>Variables y sus ámbitos</title> 
<script>
var a=10;
var b=20;
function algo(){
    var c=30;
    var b=5;
    alert(a);//10
    alert(b);//5
    alert(c);//30
}
algo();
alert(b);20
</script>
</head> 
<body>
<body>
</html>

Noten que dentro de la función la variable a está disponible, ya que se declaró fuera de la función y entonces tiene alcance global. La variable b fue redeclarada dentro de la función y su valor de ejecución para la función es, en efecto, el valor redeclarado (5). Pero fuera de la función, cuando volvemos a invocarla vemos que tiene el valor asignado en el ámbito global, es decir que la redefinición del valor solo tuvo efecto dentro de la función.


¿Pero qué hubiera sucedido si no la hubiésemos redeclarado con el prefijo var, si solo le hubiéramos asignado un nuevo valor dentro de la función sin volver a declararla?

<!DOCTYPE html>

alert(b);//20
function algo(){
    b=5;
    alert(b);//5
}
algo();
alert(b);//5
</script>
</head> 
<body>
<body>
</html>

En ese caso el valor de la variable cambia, debido a que la redefinición del valor no está hecha sobre una variable local (definida con el prefijo var dentro de la función), sino sobre la variable global.

   
Las variables globales se consideran de manera automática propiedades del objeto window y pueden ser referenciadas mediante la sintaxis de puntos, como pueden apreciar el el último alert del siguiente ejemplo:

<body>
<body>
</html>

El elemento iframe (de inline frame) permite incrustar dentro del documento html principal un documento html secundario. A efectos prácticos, es como una ventana dentro de nuestro documento que nos muestra y nos permite interactuar con una página independiente. Y dicha página puede estar en nuestro dominio o en uno diferente. 

   
Un ejemplo de uso de iframe lo encontramos a diario en las páginas que muestran videos de Youtube. O en las que muestran mapas de Google Maps en su interior.

Si ejecutan el ejemplo verán que en la página aparece una ventana hacia una página de Google Maps que muestra la ubicación de la sede de FLACSO Argentina.

Como vemos, iframe posee tag de apertura y tag de cierre y diferentes atributos que lo definen. El más importante de estos atributos es src (source) que corresponde a la ruta del archivo a desplegar. El mismo es normalmente un documento html, pero también puede tratarse de una imagen, un video, un audio, un archivo pdf, etc. Los atributos width y height permiten establecer el ancho y alto de la ventana de visualización. El atributo allowfullscreen  puesto a true permite el modo pantalla completa, que corresponde a una API avanzada de html5. Finalmente el atributo loading que aparece en el ejemplo es un atributo que indica si la carga de nuestro documento debe esperar a que cargue el iframe y luego continuar la carga de los elementos restantes de la página o avanzar con dicha carga sin esperar. Es, de todas maneras, un atributo experimental. 

   
Noten que en el ejemplo se ha añadido un estilo css para eliminar el borde del iframe, que en algunos navegadores se muestra por defecto como un recuadro de color gris.

<style>

}

Favicon

Siguiente


Un favicon, también llamado icono de página o icono de favoritos, es una imagen asociada a una web que aparece en la pestaña, barra de direcciones y en los marcadores del navegador. El primer navegador en mostrar favicons fue el Internet Explorer 5 (publicado en 1999). Internet Explorer utilizaba el formato ICO de Windows, que permite guardar varias imágenes de distintos tamaños en un único archivo  de extensión .ico.

Actualmente otros formatos son soportados, como gif, png, jpg y svg. 

Para crear un favicon podemos abrir una imagen en GIMP, escalarla a 64 x 64 px y exportarla con nombre favicon.ico. Si la guardamos con ese nombre en el directorio de publicación, que es el directorio al que apunta nuestro dominio, entonces el navegador se encargará automáticamente de mostrar ese ícono en la barra de navegación de manera automática. Si no lo podemos ubicar en ese directorio de publicación o si queremos usar otro nombre de archivo u otra extensión, entonces deberemos añadir un tag meta como el siguiente entre los tags <head></head> de nuestras páginas: 

<link rel="icon" href="imagenes/favicon.ico" type="image/x-icon" />
Para dispositivos de escritorio (desktop) y para muchos dispositivos móviles (mobile) con eso sería suficiente, pero para abarcar todo el espectro de equipos existente habría que añadir más metatags y más tamaños de imagen. Lo conveniente es utilizar para esto herramientas online que a partir de una imagen generan todo el set de íconos y todos los metadatos a añadir entre las etiquetas <heda></head>. Notarán, al ejecutar el ejemplo, que si accionan los enlaces de la barra superior se dirigirán a diferentes sectores  de la página mediante un desplazamiento brusco de las barras de scroll.

   
La propiedad CSS scroll-behavior, cuyo valor por defecto es auto, permite hacer que ese desplazamiento no se produzca de manera inmediata sino de manera suave cuando su valor se especifica a smooth.

Podemos asignar la propiedad a un elemento específico o al elemento “html” (root) si queremos que el comportamiento se replique en toda la página.

    color:#fff;
    display:inline-block;

Nomenclatura de puntos en rutas de archivos

Ir a la página principal
Siguiente


Hemos hablado de rutas de archivos cuando mencionamos las URLs. Se trata de mapas que indican la ubicación de archivos u otros recursos. Como todo sistema cartográfico, está atado a convenciones, y es por eso que en windows los diferentes componentes de las rutas (generalmente directorios) se separan con con barras invertidas (\), mientras que en Unix se utilizan las barras diagonales (/) con el mismo propósito. 


Una ruta en windows se escribe de la siguiente manera:

D:\Data\Final\archivo.jpg
En sistemas UNIX, en cambio, las rutas se expresan como sigue:
/home/cristian/carpeta1 
En desarrollo web utilizamos siempre la barra diagonal. Y sabemos que para referirnos a un archivo ubicado en el directorio actual no necesitamos incluir ninguna barra, solo su nombre: 
Imagen1.jpg
Si escribimos Imagen1.jpg como valor de la propiedad src de un tag img y el archivo gráfico se encuentra en el mismo directorio que el archivo html que contiene esa codificación, sabemos que al ejecutarlo en el navegador la imagen aparecerá en pantalla.    

Pero si la imagen estuviese no en el mismo directorio sino dentro de un subdirectorio presente en la misma ubicación que el archivo html, un subdirectorio llamado “imagenes”, entonces ya <img src=”Imagen1.jpg”> no mostraría la imagen en pantalla, debido a que el valor de src apunta a un lugar incorrecto. 

   
Si en lugar de Imagen1.jpg escribimos como src imagenes/Imagen1.jpg, entonces sí veríamos nuevamente el despliegue en pantalla de la imagen. Esto se debe a que aludimos no solo al nombre del archivo sino también a su ubicación dentro del directorio “imagenes”.  

Y, si en lugar de encontrarse dentro de “imagenes”, nuestro archivo se encontrara dentro de un subdirectorio llamado “grandes”, ubicado en el interior del directorio “imagenes”, entonces la referencia <img src=”imagenes/Imagen1.jpg”> volvería a fallar, porque nuestro mapa imagenes/Imagen1.jpg estaría incompleto. Deberíamos escribir <img src=”imagenes/grandes/Imagen1.jpg”> para que la imagen se mostrará correctamente. 

Noten que siempre el primer elemento se escribe sin barras, ya que, como dijimos, se encuentra en el mismo directorio que el html que usamos como punto de partida. Entonces, puede decirse que la ausencia de barra es una referencia al directorio actual. Pero existe otra manera explícita de referirse al directorio actual, además de la ausencia de barras: el símbolo punto:
<img src="./playa.jpg">
El punto del ejemplo anterior indica que el archivo playa.jpg se encuentra en el mismo directorio que  el archivo html que contiene ese fragmento de código. Es decir, es equivalente a lo siguiente:
<img src="playa.jpg">
Ambas expresiones son idénticas. Compruébenlo.

Así como el punto representa el directorio actual, el símbolo dos puntos (..) representa al directorio de orden superior o directorio padre del directorio actual. Si quisiéramos invocar a una imagen que se encuentra en el directorio superior al de nuestro html, la mapearíamos de la siguiente manera:

<img src="../playa.jpg">
El signo .., como dijimos, es una referencia al directorio que es padre de aquel en que se encuentra nuestro archivo html. Si quisiéramos referirnos a un archivo ubicado en el directorio superior a este último, deberíamos repetir la referencia de la siguiente manera:
<img src="../../playa.jpg">

Atributo download

El atributo html download de los elementos a (enlaces) permite abrir un cuadro de descarga para el archivo referenciado como valor de su atributo href. Si no se especifica un valor para download el archivo conservará su nombre original al ser descargado. En caso contrario asumirá el nombre que especifiquemos como valor del atributo:

Importante: las descargas solo funcionan en servidores web, no podrán testearlas de manera local. Pero pueden subir el ejemplo a su espacio de trabajo en la FLACSO para verlo en marcha.

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>descarga</title>

</head>
<body>
    <a download href="playa.jpg">Descargar</a><br>
    <a download="algo.jpg" href="playa.jpg">Descargar</a>
</body>
</html>
