Lenguaje HTML










HTML es el lenguaje de marcado que utilizamos para crear páginas web. El término “de marcado” hace referencia a las “marcas” o etiquetas que se añaden al texto presente en un documento, las cuales aportan información adicional sobre su estructura, presentación o significado.

A diferencia de los lenguajes de programación, que operan con variables y proporcionan resultados aritméticos o lógicos, los lenguajes de marcado solo añaden información contextual que es utilizada por navegadores u otros programas intérpretes.

Las marcas o etiquetas html o “tags” tienen la siguiente forma: <p>, que significa párrafo, o <a>, que significa link. La mayoría de las etiquetas tiene una “forma” de apertura y otra de cierre. Por ejemplo, un párrafo completo tiene la siguiente sintaxis: 
<p>Aquí el texto que corresponde al párrafo.</p>
Es decir, primero se ubica la etiqueta de apertura de párrafo: <p>. A continuación se escribe el contenido que corresponde al párrafo. Y finalmente se coloca la etiqueta de cierre: </p>, cuya única diferencia con la de apertura es la barra que precede a la “p” (p de paragraph en inglés).
Sin embargo, hay algunas etiquetas que no poseen una contraparte de cierre, como las etiquetas de imagen: 
<img src="foto.jpg">
Vemos que la etiqueta <img> se sustenta sin necesidad de una </img> de cierre. También podemos observar que posee un complemento, “src” (de source, en inglés), al cual se le coloca como valor la ruta correspondiente al archivo de la imagen que queramos mostrar. A estos modificadores que se añaden con un par nombre/valor a las etiquetas html se los conoce como “atributos”. Y siempre tienen la misma estructura sintáctica:
nombre + signo igual + valor entre comillas.
Además de src, hay otros atributos estándar para las imágenes, como por ejemplo width y height, que señalan el tamaño al que queremos que se visualicen: 
<img src="foto.jpg" width="200" height="100">
De la misma manera en que existen atributos estándar, podemos añadir otros personalizados: 
<img fuente="wikipedia" src="foto.jpg" width="200" height="100">

El atributo “fuente” de ese ejemplo es un atributo no estándar.

Estructura básica de una página web

Ir a la página principal
Siguiente



Toda página web tiene una estructura de etiquetas básica, que contiene a todas las otras etiquetas que conforman el sitio. Dicha estructura tiene la siguiente forma:

<!DOCTYPE html>
<html>
<head>
<title>Título de la página</title>
</head>
<body>
</body>
</html>
El primer elemento, doctype, no es en realidad una etiqueta sino una declaración que indica a los navegadores el tipo de documento que deberán interpretar. El escrito en el ejemplo corresponde a la última versión de html, que se llama html5 y es el estándar actual. Hubo versiones anteriores, que tenían tags y reglas sintácticas diferentes, e incluso hace pocos años el estándar correspondía a un híbrido entre el lenguaje de marcado xml y el html, llamado xhtml.

Suele decirse que un documento html tiene, luego del doctype, una estructura arbolar: con un tronco padre, del que parten todas las ramas hijas, y donde cada una de esas ramas hermanas da origen a otras que repiten la estructura de manera fractal.

Las ramas mencionadas también se denominan nodos y, como probablemente ya se hayan dado cuenta, cada rama se representa con una etiqueta o marca html. La rama principal, o tronco, se corresponde con el tag <html>. Y representa al nodo principal, también llamado padre o root o documentElement. De este nodo padre cuelgan 2 ramas hijas (hermanas entre sí), llamadas <head> y <body>. Y <head> a su vez tiene un nodo hijo, llamado <title>, dentro del cual siempre va escrito el título de la página. Ese texto presente en el interior de la etiqueta <title> es el dato al cual los buscadores dan más importancia a la hora de indexar las páginas webs en sus catálogos de búsqueda. También es el que vemos en las solapas del navegador cuando visitamos una página.

Los textos escritos dentro de tags html también son considerados nodos, solo que en vez de ser elementos html (ELEMENT_NODE) son nodos de texto (TEXT_NODE).

Dentro de la estructura básica podemos ver que existe otro hijo de <html>, llamado <body>

Body, o cuerpo del documento, es el elemento html dentro del cual escribiremos todo nuestro código html. En su interior alojaremos los párrafos (<p>), los links (<a>), las imágenes <img>) y todos los elementos con los que construiremos nuestros sitios web.


Semántica y Accesibilidad

Ir a la página principal
Siguiente

Antes de ver en detalle los diferentes elementos html y su funcionalidad específica, nos referiremos brevemente a 2 conceptos relacionados inherentes al lenguaje html y a la construcción de páginas web en general.

  1. La semántica: cuando hablamos de lenguaje de marcado hicimos hincapié en que las marcas aportan información sobre estructura, presentación y significado. En efecto, cada elemento html tiene una función concreta y simbólica que es importante porque aporta metainformación utilizada por navegadores web, lectores de pantalla para ciegos o personas con problemas de visión y otros dispositivos a la hora de desplegar contenidos. A esta característica funcional y de sentido de los elementos html se la conoce como “semántica”.

  2. La accesibilidad: es una característica que se mide en grados y representa la posibilidad de que una persona con cierto nivel de discapacidad pueda hacer uso de la web. La corporación encargada de definir los ítems que determinan el grado de accesibilidad de una página web es la WAI, que es una rama del World Wide Web Consortium.

Si hacemos que cada elemento html que utilicemos cumpla la función concreta para la cual ha sido creado obtendremos una web con un alto grado de accesibilidad.

Elementos de línea y elementos de bloque

Ir a la página principal
Siguiente




Cuando escribimos un documento con cualquier editor de texto, notamos que las palabras se colocan una al lado de la otra hasta ocupar el espacio horizontal disponible y que, cuando ese espacio es rebasado, la palabra que no encontró lugar en la serie se ubica en un renglón nuevo.

Hay elementos html que tienen, respecto del posicionamiento en el documento, el mismo comportamiento que las palabras: fluyen colocándose uno junto a otro hasta agotar el espacio horizontal y, cuando lo hacen, se ubican debajo del inicio de la última línea completa. Esos elementos se denominan “Elementos de línea”. Ejemplos de este tipo de elementos son las imágenes y los enlaces.

Pero hay otros elementos que, aunque no ocupen todo el espacio horizontal, fuerzan al elemento siguiente a ubicarse debajo de ellos. Estos elementos se denominan “Elementos de bloque”. Podemos señalar a los párrafos (<p>) como elementos de este tipo.

Si colocan un párrafo que contenga solo una palabra y a continuación añaden una imagen, aunque haya espacio de sobra para que la imagen se posicione junto a la palabra, comprobarán que la imagen siempre se ubicará debajo.

Los principales elementos html

Ir a la página principal
Siguiente



span: se trata de un elemento de línea, con tag de apertura y cierre, que normalmente se usa para modificar una porción de texto. Por ejemplo: 
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>Título de la página</title>
<style>
.uno span{background:yellow;}
.dos span{background:orange;}
</style>
</head>
<body>
<p class="uno">Este párrafo contiene <span>un texto</span> que voy a querer que se vea diferente</p>
<p class="dos">Este es un nuevo párrafo con <span>otro texto</span> a destacar</p>
</body>
</html>
Si abrimos un nuevo documento con Notepad++ y copiamos y pegamos en él el código anterior, lo guardamos con un nombre como test.html y luego lo ejecutamos en un navegador, veremos algo como lo que sigue:

La figura muestra cómo aparecen resaltados con colores los textos que incluimos dentro del elemento span. Para lograr el resaltado utilizamos un tag <style>, que colocamos dentro del nodo head. Más adelante profundizaremos en ese tag y en la técnica utilizada para cambiar el aspecto visual que los elementos html muestran por defecto. Por el momento sólo mencionaremos que se denomina CSS. 

Si miramos con atención el html del ejemplo, descubriremos otro tag que no mencionamos anteriormente: el tag meta. En este caso, lo usamos con el atributo charset, que le indica al navegador cómo debe mostrar algunos caracteres considerados especiales, como acentos, ñ, diéresis, etc. Si no lo incluímos es probable que esos caracteres no se vean correctamente.

Existen otros atributos que pueden añadirse al tag meta para que cambie su comportamiento. Los veremos oportunamente.

div: Se trata de un elemento de bloque, con tag de apertura y de cierre. Se lo conoce con el nombre de layer o capa y se usa normalmente como contenedor de textos y/o imágenes. La diferencia con los párrafos, además de que los párrafos sólo deberían contener texto si somos estrictos con su uso semántico, es que visualmente entre párrafos aparece por defecto un renglón vacío (ver imagen anterior).

Si en el ejemplo anterior hubiésemos utilizado <div> en lugar de <p>, ese espacio no existiría. ¡Compruébenlo!

p: Mencionamos ya varias veces a los párrafos. Son elementos de bloque, con tag de apertura y de cierre. Semánticamente están pensados como contenedores de texto que agrupan oraciones que hablan de un mismo tema y finalizan con un punto aparte.

a: Link o enlace (“a” de anchor). Son elementos de línea, con tag de apertura y cierre. Su misión es redirigir a una nueva página, a otro lugar dentro del documento o a un archivo descargable. Pueden contener texto o imágenes. Veamos un ejemplo: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Título de la página</title> 
</head> 
<body> 
<p>Pulsa <a target="_blank" href="https://www.flacso.org.ar/">aquí</a> para abrir en una nueva ventana la página de la Facultad</p> 
<div> 
    Haz clic en el logo para abrir la página de la universidad en la misma ventana: 
    <div><a href="https://www.flacso.org.ar/"><img src="https://www.flacso.org.ar/wp-content/themes/flacso_v2/img/header/logo_flacso.jpg" width="150"></a></div> 
</div> 
</body> 
</html>

Utilicen Notepad++ para ver el ejemplo en un navegador web. Debe lucir así:

En el primer caso, el elemento a tiene en su interior un nodo de texto: la palabra “aquí”.  También posee 2 atributos. Uno llamado href, cuyo valor corresponde a la URL a la que queremos redirigir cuando alguien hace clic en el conjunto. Y otro atributo, llamado target, que indica si la redirección debe hacerse utilizando la misma ventana o una nueva. Si utilizamos como valor de target la clave _blank, se abrirá en una nueva ventana. Y si usamos _self o no definimos un target, la página nueva se abrirá en la misma ventana donde se encuentra la página que contiene el enlace.

Vemos en el segundo enlace que el elemento a posee un nodo hijo de tipo imagen. Noten que colocamos este último enlace dentro de un div. Si no lo hubiéramos hecho, ya que los enlaces son elementos de línea, se habría posicionado a continuación del texto en lugar de aparecer debajo del mismo. También queríamos mostrarles que es posible anidar elementos: en el ejemplo verán que usamos un div dentro de otro div, lo cual es perfectamente válido. Es posible el anidamiento de la mayoría de los elementos HTML, aunque hay excepciones que son bastante intuitivas. Por ejemplo, no es posible colocar un enlace dentro de otro, lo que resulta lógico porque ¿a cuál otorgaría el navegador prioridad a la hora de redireccionar? Por una razón semejante y también por no tener tag de cierre, está vedado el anidamiento de imágenes.

br: Justamente en el ejemplo anterior vimos que para forzar a un elemento de línea a posicionarse debajo de otro elemento del mismo tipo teníamos que envolverlo en uno de bloque (utilizamos un tag div, en ese caso). No es eso realmente lo más práctico y quizá tampoco lo más acorde con la semántica, ya que priorizamos el comportamiento visual de los elementos por encima de su función específica. El elemento br se utiliza para provocar un salto de línea. Solo tiene tag de apertura y, como todo elemento que se ubique detrás de él en el documento se irá obligatoriamente al renglón siguiente es, en efecto, un elemento de bloque.

Veamos el ejemplo anterior usando el elemento br: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Título de la página</title> 
</head> 
<body> 
<p>Pulsa <a target="_blank" href="https://www.flacso.org.ar/">aquí</a> para abrir en una nueva ventana la página de la Facultad</p> 
<div> 
    Haz clic en el logo para abrir la página de la universidad en la misma ventana: 
    <br> 
    <a href="https://www.flacso.org.ar/"><img src="https://www.flacso.org.ar/wp-content/themes/flacso_v2/img/header/logo_flacso.jpg" width="150"></a> 
</div> 
</body> 
</html>

No hay diferencia con el modelo anterior al verlo en un navegador. ¡Compruébenlo!

i: Es un elemento de línea con tag de apertura y tag de cierre. La i es por italic. Su función es semejante a la del tag span, pero más específica: muestra en cursiva el texto envuelto en sus tags.

b: Es un elemento de línea con tag de apertura y tag de cierre. La b es por bold. Su función es semejante a la del tag span, pero más específica: muestra en negrita el texto envuelto en sus tags.

u: Es un elemento de línea con tag de apertura y tag de cierre. La u es por underline. Su función es semejante a la del tag span, pero más específica: muestra subrayado el texto envuelto en sus tags.

img: Al igual que con los párrafos, ya hemos hablado un poco de este elemento. Se trata de un elemento de línea, sin tag de cierre. Su función es mostrar en pantalla la imagen correspondiente al archivo cuya ruta indiquemos en su atributo src. (Img por image y src por source, en inglés). Como vimos previamente, además del atributo src es posible añadir los atributos de control de tamaño width y height, que reciben como valor el número de píxeles correspondientes al ancho y alto respectivamente. Si la proporción con que definimos esos valores no coincide con la relación alto/ancho real del archivo de origen, veremos en pantalla una imagen deformada.

Si sólo definimos uno de los dos atributos, por ejemplo width, el otro (height) será calculado automáticamente por el navegador para mostrar la imagen en su proporción original. (Esto lo vimos cuando ejecutamos el archivo que contenía un link de texto y otro de imagen).

Las imágenes tienen otro atributo cuyo añadido es obligatorio si queremos que nuestra página web sea accesible, denominado alt (de alternative text). Se utiliza para añadir una breve descripción de la imagen que sea útil a los lectores para invidentes. Ejemplo: 
<img src="https://www.flacso.org.ar/wp-content/themes/flacso_v2/img/header/logo_flacso.jpg" width="150" alt="Logo de la FLACSO">
ul, ol, li: Son todos elementos de bloque, con tags de apertura y cierre, cuya función es generar listas no ordenadas u ordenadas.
Un ejemplo de lista no ordenada tendría la siguiente estructura:
<ul> 
    <li>Primer elemento de la lista</li> 
    <li>Segundo elemento de la lista</li> 
    <li>Tecer elemento de la lista</li> 
</ul>
 También suele verse en barras de navegación: 
<ul> 
    <li><a href="https://www.flacso.org.ar/noticias/">Noticias</a></li> 
    <li><a href="https://www.flacso.org.ar/calendario/">Calendario</a></li> 
    <li><a href="https://www.flacso.org.ar/biblioteca/">Biblioteca</a></li> 
</ul>
Si probamos ambos ejemplos en un navegador veremos que cada elemento de la lista aparece precedido por una viñeta que, por defecto es un dico oscuro, pero que puede modificarse utilizando un atributo del elemento li llamado type, que puede adoptar 3 valores: disc, circle o square.
<ul> 
    <li type="disc">uno</li> 
    <li type="circle">dos</li> 
    <li type="square">tres</li> 
</ul>
Si en lugar de envolver los elementos li en un tag ul (de unordered list), los envolvemos en un tag ol (de ordered list), obtenemos una lista en la cual las viñetas son sustituidas por números: 
<ol> 
    <li>uno</li> 
    <li>dos</li> 
    <li>tres</li> 
</ol>
Es posible que en lugar de números arábigos aparezcan números romanos o secuencias alfabéticas. Esto se controla mediante el atributo type del elemento ol, que admite los siguientes valores: 1, a, A, i e I:
<ol type="I"> 
    <li>uno</li> 
    <li>dos</li> 
    <li>tres</li> 
</ol>

Formularios

Siguiente




Los formularios son conjuntos de elementos especiales, denominados elementos de formulario (html form elements), contenidos dentro un tag padre llamado form.

form:  es un elemento de bloque, con tag de apertura y cierre. Su función es tomar la información de algunos elementos que veremos a continuación (y que se encuentran en su interior) y enviarla a una página de proceso para que esta realice una tarea.

Seguramente han visto páginas web que poseen un área de contacto, con campos para escribir y un botón para enviar la información. Al presionar el botón de envío, el cual es un elemento de formulario, lo que están haciendo es pedirle a un tag form que recoja todos los datos escritos en los campos y los envíe a una página que reúna toda esa información y se la envíe por mail al administrador de la página.

input: es un elemento de línea sin tag de cierre, que puede asumir un valor que los usuarios generalmente pueden cambiar. Suele tener un atributo name, que es utilizado por las páginas de proceso como identificador.

Hay varios tipos de input, que se diferencian unos de otros por su atributo type. Y opcionalmente pueden tener un atributo value, que permite que definamos en ellos un valor inicial. Veamos un ejemplo correspondiente a algunos inputs de tipo texto: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Formularios</title> 
</head> 
<body> 
<h1>Ejemplo de formulario</h1> 
<form> 
<input type="text" name="nombre"> <input type="text" name="apellido"><br><br> 
<input type="text" name="anio_de_inicio" value="2021"><br><br> 
<input type="text" name="nombre_del_curso" placeholder="Nombre de este curso"> 
</form> 
</body> 
</html>

Hay varias cosas para observar en el ejemplo. Si lo ejecutan en el navegador, verán que los 2 primeros campos se ubican uno al lado del otro. Esto ocurre porque, como dijimos, los inputs son elementos de línea. Los restantes se ubican debajo debido al uso de tags br.

El tercer input tiene un valor inicial, que es el que escribimos como valor del atributo value. Y también posee una peculiaridad en el valor del atributo name: en lugar del término año de inicio, que es el que claramente correspondería, escribimos anio_de_inicio. Es decir, sustituimos la ñ y los espacios en blanco. Esto se debe a que es recomendable que los valores correspondientes al atributo name no posean espacios, vocales acentuadas y, en general, los mismos caracteres que sugerimos no utilizar en los nombres de archivo.

Finalmente, en el cuarto input podemos ver que, aunque no tiene un valor escrito, posee en su interior un texto informativo, coincidente con el que está definido en el atributo placeholder. Se trata de una herramienta que puede ser útil en algunas circunstancias.

A continuación, mostraremos otros tipos de inputs: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Formularios</title> 
</head> 
<body> 
<h1>Ejemplo de formulario</h1> 
<form> 
<input type="text" name="nombre"><br><br> 
<input type="password" name="clave"><br><br> 
<input type="checkbox" name="es" value="español"> español <input type="checkbox" name="en" value="inglés"> ingles<br><br> 
<input type="radio" name="origen" value="nacional"> nacional <input type="radio" name="origen" value="importado"> importado 
</form> 
</body> 
</html>
Nuevamente aquí hay varios puntos a señalar: el input 1 es de tipo text, es decir, idéntico a los que ya vimos. El siguiente, en cambio, es de tipo password y, aunque cumple la misma función que el anterior y que cualquier input (generar un dato de nombre idéntico a su atributo name y valor igual a su atributo value o a lo que se escriba en él), los caracteres que ingresemos en su interior serán reemplazados por puntos o asteriscos como medida de seguridad.


A continuación, vemos 2 inputs de tipo checkbox, los cuales se utilizan para definir opciones de temática afín que pueden seleccionarse de manera simultánea. En el ejemplo, una persona puede seleccionar uno o ambos idiomas. Notemos que el name de cada checkbox es diferente, lo mismo que su value.

Finalmente se nos presentan 2 inputs de tipo radio, que se utilizan para definir conjuntos de opciones excluyentes, es decir, que sólo podemos elegir una de ellas. Notemos que en este caso ambos inputs tienen el mismo atributo name y diferentes values. Si el atributo name fuera diferente, el conjunto se rompería y no funcionaría de manera adecuada.

Algo que no es muy correcto del ejemplo anterior es que para seleccionar un checkbox o un radio tenemos que apuntar con el puntero del mouse o con el dedo de manera precisa. Esto atenta contra la “usabilidad” o comodidad del usuario y debe mejorarse.


Existe un atributo especial, aplicable a cualquier elemento html, llamado id. Tiene un par de requisitos: su valor debe comenzar con un carácter alfabético, no debe contener espacios ni caracteres considerados especiales (acentos, ñ, diéresis, etc.). Y no debe repetirse en el documento, ya que oficia como identificador único del elemento que lo posee. Cuando veamos estilos CSS y JavaScript veremos en profundidad su importancia. Lo mencionamos ahora porque también es utilizado en un tag  html que permite añadir etiquetas a campos de formulario y que, además, interactúa con ellos. Se trata del elemento label.

label: es un elemento de línea, con tag de apertura y cierre, cuya función es añadir información y funcionalidad de selección a controles de formulario. Puede utilizarse de 2 maneras: envolviendo al elemento a manera de contenedor, o de manera externa, con un atributo for cuyo valor sea igual al id del elemento de formulario con que vaya a interactuar. Ejemplo:
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Formularios</title> 
</head> 
<body> 
<h1>Ejemplo de formulario</h1> 
<form> 
<label>Nombre:<br> 
<input type="text" name="nombre"> 
</label> 
<br><br> 
<label for="pass">Contraseña:</label><br> 
<input type="password" name="clave" id="pass"> 
<br><br> 
<label><input type="checkbox" name="es" value="español"> español</label> 
<input type="checkbox" name="en" value="inglés" id="en"> <label for="en">inglés</label><br><br> 
<label><input type="radio" name="origen" value="nacional"> nacional</label> <input id="importado" type="radio" name="origen" value="importado"> <label for="importado">importado</label> 
</form> 
</body> 
</html>
En el ejemplo hay muchos ítems para analizar. Para el primero de cada par de elementos utilizamos la modalidad contenedora de la etiqueta label. Es decir, introdujimos en ella el campo y le añadimos un texto informativo a manera de etiqueta. Para el segundo elemento de cada par utilizamos en cambio la modalidad de afectación externa, mediante la utilización del atributo for. Es decir, añadimos al campo a modificar un id con un valor determinado y al tag label le asignamos un atributo for de valor idéntico. Nótese que la posición de estos labels externos es indistinta: en el ejemplo los ubicamos delante en los input de texto o contraseña y detrás en los checkboxes y radios. También es indiferente la ubicación de los atributos html: es indistinto que el id esté delante del name o del value y viceversa.

En el lenguaje html no importa el orden de los atributos, solo es obligatorio que sus nombres comiencen con caracteres alfabéticos y no posean caracteres extraños ni espacios, y que sus valores respeten las normas específicas de cada elemento.

Tampoco tiene importancia que el valor de un id coincida con el del name del mismo elemento: mientras el id no se repita en el documento, comience con una letra y no tenga espacios ni caracteres extraños diferentes a guiones medios o bajos, será correcto.
Si hacemos clic sobre las palabras definidas dentro de los tags labels veremos que el campo con el que interactúa el label es seleccionado o enfocado, lo cual es una gran mejora de usabilidad respecto al formulario correspondiente al anterior ejemplo.

button: es un elemento de línea con tag de apertura y cierre, perteneciente a la familia de los elementos de formulario. Su función es enviar a una página de proceso el set de datos correspondiente al formulario al ser presionado, o ejecutar alguna instrucción JavaScript. Veamos un ejemplo: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Formularios</title> 
</head> 
<body> 
<h1>Ejemplo de formulario</h1> 
<form> 
<label>Nombre:<br> 
<input type="text" name="nombre"> 
</label> 
<br><br> 
<label>Apellido:<br> 
<input type="text" name="apellido"> 
</label> 
<br><br> 
<label>Nick:<br> 
<input type="text" name="nick"> 
</label> 
<br><br> 
<button type="submit">Enviar</button> 
</form> 
</body> 
</html>
Si ejecutan en un navegador el código anterior con la ayuda de Notepad++ y luego de completar el formulario presionan el botón enviar, verán que el formulario es enviado. Van a notar que lo que habían escrito en los campos desapareció y que la URL de la barra de direcciones del navegador se convirtió a algo semejante a esto:

C:/wamp64/www/flacso/6.html?nombre=Andr%C3%A9s&apellido=Fern%C3%A1ndez&nick=Panino
Noten que luego del nombre del archivo html aparece un signo de interrogación y a continuación una cadena de caracteres. Si observamos el conjunto con atención descubriremos que tiene la siguiente estructura:
Archivo html  ?  name del primer campo  =  lo que escribimos en el primer campo & name del segundo campo = lo que escribimos en el segundo campo & name del tercer campo = lo que escribimos en el tercer campo
Es decir, los datos del formulario están presentes en la url, separados del nombre del archivo por un signo de interrogación y distribuidos en pares nombre=valor, delimitados por un signo &. Al mismo tiempo podemos notar que los datos no se ven en la url como los habíamos escrito, sino que aparecen adaptados a la codificación URL, denominada escape hexadecimal.


La misión fundamental de los formularios, como mencionamos anteriormente, es enviar datos a páginas de proceso para que estas hagan alguna operación con ellos. En nuestro ejemplo no definimos un atributo action al tag form. Es en ese atributo donde se especifica el nombre de la página de proceso. Y si no se define, la información se envía a la misma página que contiene el formulario.

Otro atributo del tag form que afecta la manera de envío se llama method. Si no lo determinamos el form adopta de manera automática el valor de GET, que indica que los datos deben transmitirse por la URL, como en nuestro ejemplo. El valor que asume la mayoría de los formularios de las páginas que visitamos a diario es POST, que ocasiona que la información se propague a través de mecanismos menos visibles.

Veremos a continuación un ejemplo de envío a una página de proceso programada en javascript. No se preocupen ahora por entender el código, no es necesario en esta etapa:
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>Procesar Formulario</title> 
<script> 
onload=function(){ 
    if(location.search.length>1){ 
        var pares=location.search.substr(1).split('&'); 
        var html='',par,i,l; 
        for(i=0,l=pares.length;i<l;i++){ 
            par=pares[i].split('='); 
            html+='<b>'+decodeURIComponent(par[0])+':</b> '+decodeURIComponent(par[1] || '-').replace(/\+/g, " ") +'<br>'; 
        } 
        document.body.innerHTML+=html; 
    } 
} 
</script> 
</head> 
<body> 
</body> 
</html>
Con ayuda de Notepad++ copien el código anterior en un documento y guárdenlo con el nombre pagina-de-proceso.html.

En el mismo directorio en que guardaron pagina-de-proceso.html, guardarán otro archivo con el siguiente código y luego lo ejecutarán en el navegador: 
<!DOCTYPE html>

<body>

</label>

Notarán que es casi idéntico a otro ejemplo que hemos visto, solo que en esta ocasión hemos incluído el atributo action del tag form y le hemos puesto como valor la ruta al archivo de proceso que creamos en el paso anterior. Si completan el formulario y lo envían, serán redirigidos a la página de proceso que indicamos como valor para action, la cual en este ejemplo toma los datos del formulario y los muestra en pantalla.

Otras páginas de proceso enviarían esos datos por mail o los guardarían en una base de datos o en archivos de texto o realizarían otra operación en función de cómo sean programadas.

 
Habíamos mencionado que los botones pueden, además de enviar formularios, ejecutar ciertas instrucciones javascript. Para eso deben tener un atributo type con valor diferente del que estuvimos utilizando hasta ahora.

Si retornan a los ejemplos anteriores, advertirán que usamos submit, que es la clave con que se indica al navegador que envíe el formulario cuando el botón sea presionado. También es el valor por defecto, lo que significa que, si no hubiésemos definido el atributo type, el botón habría actuado de la misma manera en que lo hizo.

<!DOCTYPE html>

Si le cambian el valor de type a submit o directamente eliminan el atributo type, comprobarán que luego de ejecutar el código javascript el formulario es enviado y se recarga la página, cosa que no ocurre con el type button.

select: es un elemento de línea con tag de apertura y cierre, que contiene en su interior a uno o más subelementos option. Al igual que los inputs de tipo radio, se utiliza para seleccionar un único valor de un set de opciones. La decisión de usar select en lugar de radio normalmente está ligada a la cantidad de opciones: si son muchas, por eficiencia, legibilidad y espacio es preferible usar selects. Además, los selects muestran las opciones que comienzan con la letra correspondiente a la tecla que pulsemos sobre ellos, lo que constituye una gran ventaja de usabilidad, especialmente si las opciones que contienen se encuentran ordenadas alfabéticamente.

<html>

<meta charset="utf-8"> 
<title>Formularios</title> 
</head>

También que el name que identificará al dato a enviar a dicha página lo añadimos al tag select. Si en cambio eligen la segunda opción, que es la que no tiene definido un atributo value, verán que el valor que asume la variable enviada corresponde al texto de la opción seleccionada.

Un uso avanzado y poco frecuente del elemento select es la selección múltiple. ¡Busca información al respecto!

Estilos CSS

Siguiente

CSS, sigla en inglés de Cascade Style Sheet, que se traduce como Hojas de Estilo en Cascada, es un lenguaje que permite modificar el aspecto de un documento escrito en lenguaje de marcado. Forma parte de una estrategia de desarrollo que tiene como objetivo separar la forma del contenido.

La palabra cascada se debe a que el lenguaje utiliza un mecanismo llamado herencia, el cual permite asignar reglas de presentación a los nodos superiores, las cuales automáticamente se propagan a los inferiores. Al mismo tiempo, ofrece la posibilidad de modificar o sobreescribir estas reglas de manera jerárquica (escalonada) a fin de lograr especificidad.

Un ejemplo: si con el lenguaje establecemos (ya veremos cómo) que el elemento body posea un color de fondo rojo, a partir de esa definición todos los elementos que estén dentro del tag body, ya sean descendientes directos o indirectos, tendrán un color de fondo idéntico al de su ancestro. Para que cualquier elemento derivado de body tenga un color de fondo diferente, tendremos que escribir, a continuación de la primera, una nueva instrucción en la que identifiquemos de manera clara el elemento modificar y determinemos el color que deseemos que adopte, sabiendo de antemano que dicha instrucción afectará de manera recursiva a todos los nodos contenidos en el elemento en cuestión.

Este modo de funcionamiento, que nos concede la opción de ir de lo general a lo particular, también nos obliga a ser precavidos ya a que una instrucción posterior puede modificar o anular a otra escrita previamente.

En CSS el orden es muy importante.

Formas de añadir estilos CSS

Ir a la página principal
Siguiente


Hay varias maneras de incluir CSS en un documento. Las más conocidas son las siguientes:

  1. En el mismo documento.
  2. En cada elemento HTM.
  3. Mediante un documento externo.


En el mismo documento

Este mecanismo de inclusión de estilos requiere el uso de un elemento html llamado style, el cual posee tag de apertura y cierre. Normalmente se lo ubica en el head del documento. Veamos un ejemplo: 
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>CSS: inclusión usando el tag style</title>
<style>
body{
    background:red;
}
</style>
</head>
<body>
</body>
</html>

Verán que luego de los tags meta y title hemos ubicado un elemento style. Y en su interior hemos escrito una instrucción que analizaremos más adelante pero que, como podrán apreciar intuitivamente o mediante la ejecución del archivo, sirve para cambiar el color de fondo del elemento body.

En cada elemento HTML


Podemos añadir a cualquier elemento html un atributo style y asignarle como valor una o más instrucciones css. Veamos un ejemplo: 
<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>CSS: inclusión inline</title>
</head>
<body style="background:red;">
<div style="position:absolute; width:100px; height:100px;background:yellow; top:50%; left:50%; transform:translate(-50%,-50%);border-radius:50%;"></div>
</body>
</html>

Ya veremos en detalle la sintaxis correspondiente a las instrucciones CSS, pero adelantaremos que se trata de pares propiedad/valor separados por punto y coma. En el ejemplo hemos asignado solo una instrucción al tag body, pero hemos añadido ocho al elemento div que tiene incluído.

Esta manera de incluir estilos CSS se denomina “inline” y no está considerada como una buena práctica, ya que mezcla demasiado las capas de contenido y presentación, lo que siempre aporta complicaciones al desarrollo y al mantenimiento de la página. Sin embargo, hay un caso en que su uso es prácticamente obligatorio si queremos que el producto funcione correctamente: los mails en formato HTML.

Mediante un documento externo

La siguiente manera de incluir estilos CSS es la más apreciada por ser la que mejor cumple con el criterio de separación forma/contenido.


Se vale de un elemento html denominado link, que posee 2 atributos. El primero, de nombre rel (por relationship), indica la relación entre el documento html anfitrión y el documento a vincular y tiene como valor la clave stylesheet. El segundo, llamado href (hipertext reference), lleva como valor la ruta al archivo que contiene las instrucciones CSS.

Copien y peguen en un nuevo documento de Notepad++ el siguiente código y guárdenlo en un directorio con el nombre estilos.css: 
body{ 
    background:blue; 
    color:#FFF; 
    font-family:Arial, Helvetica, sans-serif; 
    font-size:16px; 
}
A continuación, en un nuevo documento añadan este fragmento: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS: inclusión mediante archivo externo</title> 
<link rel="stylesheet" href="estilos.css"> 
</head> 
<body> 
Este es un ejemplo de inclusión de CSS con un archivo externo. 
</body> 
</html>

Guárdenlo con cualquier nombre y extensión html en el mismo directorio en que se encuentre estilos.css, y ejecútenlo en el navegador.

Un documento puede tener más de una hoja de estilos: basta añadir un elemento link para cada una de ellas y tener en cuenta que si algún estilo definido en una de ellas colisiona con el definido en otra, prevalecerá el que corresponda a la última. Esto ocurre porque el navegador siempre interpreta el código por orden de aparición, aunque este se encuentre en archivos externos.

Aunque no sea una práctica habitual, debemos mencionar que es posible usar, de manera simultánea y en un mismo documento, todas las maneras de inclusión mencionadas.

Sintaxis CSS

Ir a la página principal
Siguiente

Las instrucciones en CSS se denominan declaraciones. Cada declaración está constituida por un par propiedad/valor. Las propiedades son características visuales, como por ejemplo el color de fondo, el tamaño, la fuente a utilizar, la posición, etc. Y los valores son los estados que queremos que asuman esas propiedades cuando el navegador muestra la página. Un conjunto de declaraciones luce de esta manera:
color:black; background:#FF0000; margin:10px;
El signo que se utiliza en CSS para separar una declaración de otra es el punto y coma. Con base en eso podemos afirmar que en la serie anterior hay 3 declaraciones, que son las siguientes:

  1. color:black
  2. background:#FF0000
  3. margin:10px

A su tiempo explicaremos con más detalle cada propiedad y cada valor. Ahora nos centraremos en la sintaxis y en una comprensión parcial de los componentes.


En toda declaración CSS la propiedad siempre precede al valor.

En el primer ítem de la muestra, la propiedad es color (que en CSS determina el color del texto). Y el valor que se le asigna a esa propiedad es black.

Noten que entre la propiedad color y el valor black hay dos puntos. Esos dos puntos en CSS funcionan como un operador de asignación, es decir, a la propiedad color se le asigna el valor black. En otras palabras, se pretende que el texto se muestre en color negro.

Dijimos que lo que en cada declaración separa la propiedad del valor es el signo “:” y que el orden propiedad/valor es inalterable o no conmutativo. Es claro entonces que en la segunda declaración tenemos una propiedad llamada background (que en CSS define el fondo de uno o más elementos) a la que se le está adjudicando el valor #FF0000 (rojo en sistema hexadecimal).

Finalmente, la última declaración alude a la propiedad margin, que refiere a la distancia que separa a un elemento de su contenedor o de nodos adyacentes. El valor asignado a esa propiedad es, en este caso, de 10px: un número y una unidad de longitud.

Cuando mencionamos los estilos inline dijimos que, como valor del atributo style a incluir en un elemento html para modificar su aspecto, podíamos agregar una o más instrucciones CSS. Lo que en realidad añadimos en ese espacio es un conjunto de longitud variable de declaraciones CSS, semejante al que acabamos de ejemplificar.

Consideren que hasta ahora hablamos de las declaraciones, pero no de los elementos HTML afectados por ellas.

En el caso de los estilos inline, es bastante claro que el elemento al que se le añade el atributo style es el alcanzado por el conjunto de declaraciones.

Pero cuando empleamos un documento externo o un tag style, necesitamos alguna estructura que permita establecer de manera clara la identidad del nodo al cual vamos a asignarle un conjunto de declaraciones dado. Esa estructura se llama regla y tiene la siguiente forma: 
body{background:orange; color:#000000; font-family:Arial, Helvetica, sans-serif;}
Lo primero que notamos en esa estructura es que se hace referencia al tag body. Luego, que entre llaves hay un conjunto de declaraciones css semejante a los que hemos conocido.


El componente que precede a la primera llave (body en este caso) se denomina selector. Y todo el fragmento que lo acompaña (llaves incluidas) se conoce como bloque de declaraciones.

Dentro de un tag style o de un documento css podemos introducir una cantidad indeterminada de reglas. Pueden separarse mediante espacios, pero lo normal es escribir una debajo de otra. Dentro de los bloques, las declaraciones pueden ordenarse como se desee. Y las reglas pueden también definirse de manera arbitraria, teniendo siempre presente que, en caso de colisión, será siempre la última la que prevalecerá.

Hemos mencionado que el primer elemento de una regla css se denomina selector.

Existen selectores básicos y avanzados. Los básicos son los siguientes:
  1. Selector de tipo o etiqueta.
  2. Selector de clase.
  3. Selector de id.

Selector de tipo o etiqueta

Los selectores de tipo o etiqueta se definen con nombres de elementos html, como img, body o p, por ejemplo. E indican que todos los elementos del tipo señalado asuman las propiedades y valores consignados en el bloque de declaraciones que preceden.
Vean el cuadro siguiente: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style> 
body{ 
    background:gray; 
} 
p{ 
    background:red; 
    color:#FFFFFF; 
} 
a{ 
    background:orange; 
    color:#000000; 
} 
</style> 
</head> 
<body> 
    <div><p>párrafo dentro de div</p></div> 
    <p>párrafo suelto</p> 
    <a href="#">enlace</a> 
</body> 
</html>
Si ejecutan ese código en un navegador, comprobarán que todos los párrafos adquieren el estilo indicado por el selector p, los enlaces el señalado por a y el cuerpo del documento los asignados al selector body.

Selector de clase

El término clase en programación se utiliza para referirse a un mecanismo que brinda la posibilidad de definir un conjunto de propiedades susceptible de ser asignado a diferentes objetos. Es decir: se define una vez y se reutiliza las veces que sean necesarias.

En css basta con añadir a uno o más elementos html un atributo class con un valor determinado, para que dichos elementos asuman las propiedades que les asignemos.

Vean el siguiente ejemplo: 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style> 
body{ 
    background:gray; 
} 
.rojo{ 
    background:red; 
    color:#FFFFFF; 
} 
.naranja{ 
    background:orange; 
    color:#000000; 
} 
</style> 
</head> 
<body> 
    <div><p class="naranja">párrafo dentro de div</p></div> 
    <p class="rojo">párrafo suelto</p> 
    <a class="rojo" href="#">enlace</a> 
</body> 
</html>
Noten que la primera regla contiene un selector de tipo y las siguientes presentan selectores que comienzan con un punto, seguido por una palabra clave.

Se los conoce como selectores de clase, y hacen referencia a elementos html con atributos class cuyos valores coincidan con la palabra que sucede al punto con que empiezan.

El primer párrafo tiene un atributo class de valor naranja (sin punto), eso quiere decir que será afectado por la regla cuyo selector sea .naranja.

El segundo párrafo tiene un class con valor rojo, al igual que el enlace. Eso significa que ambos serán afectados por la regla correspondiente al selector .rojo.

Comprobarán, al ejecutar el ejemplo en un navegador, que el último párrafo y el enlace comparten el mismo estilo.

Un mismo elemento puede ser modificado por varias reglas. Solo hay que escribirlas como valor del atributo class y separarlas con espacios. 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style> 
body{ 
    background:gray; 
} 
.uno{ 
    background:red; 
    color:#FFFFFF; 
} 
.naranja{ 
    background:orange; 
    color:#000000 
} 
</style> 
</head> 
<body> 
    <div><p class="naranja uno">párrafo dentro de div</p></div> 
    <p class="uno naranja">párrafo suelto</p> 
    <a class="uno" href="#">enlace</a> 
</body> 
</html>

Noten que en el primer párrafo escribimos naranja uno y en el segundo invertimos el orden. Como comprobarán si ejecutan el código en un navegador, eso resulta indiferente: el estilo de ambos elementos resulta idéntico, pese a que en las clases que los atañen la propiedad background colisiona.

Como ya hemos señalado, la definición que prevalece es la última que se declara, que en este caso es orange. Prueben qué sucede si intercambian el orden de las reglas con selectores .uno y .naranja. Verán que siempre vence la que aparece al final.

En la regla de selector .naranja mostramos cómo es posible omitir el punto y coma al final de la última declaración: en esa ubicación no se justifica su presencia ya que su función de separador no es necesaria.


Selector de id

Nos referimos al atributo id cuando trabajamos con labels y elementos de formulario.  Dijimos, en aquella oportunidad, que se trataba de un atributo que podía ser incluido en cualquier elemento html y que su valor debía comenzar siempre con un carácter no numérico, que no debía contener espacios ni caracteres especiales a excepción de guiones medios o bajos y, fundamentalmente, que no podía repetirse en el mismo documento.

Que su valor sea único se debe a que se utiliza para referenciar de manera inequívoca al elemento que lo posee.

En CSS los selectores de id se escriben con un carácter # (numeral) seguido del id correspondiente a un elemento html. 
<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style> 
body{ 
    background:gray 
} 
#uno{ 
    background:red; 
    color:#FFFFFF 
} 
#dos{ 
    background:#0000FF; 
    color:#FFFFFF 
} 
.enlace{ 
    background:orange; 
    color:#000000 
} 
</style> 
</head> 
<body> 
    <div><p id="uno">párrafo dentro de div</p></div> 
    <p id="dos">párrafo suelto</p> 
    <a class="enlace" href="#">enlace</a> 
</body> 
</html>
Pueden ver en el ejemplo cómo cada selector de id afecta únicamente a los elementos identificados con la clave escrita detrás del signo numeral.

Las primeras 2 reglas tienen selectores de tipo o etiqueta. La tercera, en cambio, contiene un selector que combina la clave div, un espacio en blanco y la clave p. Se trata de un selector descendente. Para comprender el alcance de este tipo de selectores conviene leerlos de derecha a izquierda. Entonces, tenemos una p, que podemos traducir como “todos los elementos de tipo párrafo”, luego un espacio, que significa “contenidos en” y por último la palabra div que interpretamos como “elementos de tipo div”. Es decir: todos los párrafos contenidos en divs.

En el ejemplo tenemos solo un tag div con un párrafo en su interior. Según la regla debería aparecer con un fondo rojo. ¡Compruébenlo! La cuarta regla tiene 3 palabras clave separadas por espacios: div p a. La sexta regla también posee un selector descendente: #uno span. Debe leerse: todos los elementos de tipo span contenidos en un elemento que tenga un atributo id con valor uno.

Dentro del tag body hay un párrafo que posee un id de valor uno con varios elementos de tipo span en su interior. El navegador les mostrará que cada span se presenta con un fondo de color violeta, tal como indica la regla. Noten que todos los span, sin importar que sean descendientes directos o no, como es el caso del que se encuentra dentro de un enlace, son alcanzados.

background: la propiedad background determina cómo debe verse el fondo de un elemento html. Puede contener un color o una imagen o ambos. Los colores en css pueden definirse de varias maneras:

  1. Con palabras clave: transparent, black, white, orange, red, yellow, green, etc.
  2. Con las funciones rgb() y rgba()
  3. Con las funciones hsl() y hsla()
  4. Mediante codificación hexadecimal
La primera opción utiliza como claves los nombres de los colores en inglés. Aquí pueden ver una lista que los enumera, junto con su valor hexadecimal: https://developer.mozilla.org/es/docs/Web/CSS/color_value

Las funciones rgb (por las iniciales correspondientes a red, green, y blue) aluden a la forma de representación de colores basada en la combinación de 3 colores básicos: rojo (red), verde (green) y azul (blue). La sintaxis para una declaración correspondiente a la propiedad color luce de esta manera:

color: rgb(255,0,0);

Los valores separados por coma dentro del paréntesis se llaman parámetros. El primero corresponde a la proporción de rojo de la mezcla, el segundo a la de verde y el tercero a la de azul. Cada valor puede definirse en rangos de 0 a 255 para números enteros, o de 0 a 100 si definimos cada valor en porcentaje. La siguiente declaración es exactamente igual a la anterior:

color:rgb(100%,0%,0%);

La función rgba es semejante a la anterior, pero usa un cuarto parámetro llamado alpha, referido al grado de opacidad. Es un número entre 0 y 1 (cero es totalmente transparente y 1 es totalmente opaco).

<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style>

    background:rgb(50%,50%,50%) 
} 
#verde{ 
    background:rgba(0,255,0,0.5); 
    width:100px; 
    height:100px; 
    position:absolute; 
    top:50px; 
    left:50px; 
}

    background:rgba(255,255,0,0.5); 
    width:100px; 
    height:100px; 
    position:absolute; 
    top:100px; 
    left:100px; 
} 
</style>

Las funciones hsl (por las iniciales pertenecientes a hue, saturation y ligthness) permiten definir colores con base en tres parámetros: matiz (hue), saturación (saturation) y luminosidad (lightness) . El matiz es un número de 0 a 360, que indica el color que aparece en ese grado de la siguiente rueda comática:


El 0 corresponde al rojo, el 120 a verde y el 240 al azul (el radiovector avanza en sentido horario).

La saturación refiere a la pureza del color y se define en porcentaje: 100% indica un color saturado y puro, mientras que porcentajes más bajos hacen que el color vire a gris.

La luminosidad también se define en porcentaje. 0% es el grado menor y provoca que el color vire al negro. 100% es el grado de luminosidad mayor y hace que el color vire a blanco. Un valor de 50% no produce cambios.

La siguiente declaración es equivalente a color:red y a color:rgb(255,0,0):

color: hsl(0, 100%, 50%);


hsla funciona igual que hsl, pero con el añadido del parámetro a, de alpha, que trabaja igual que en en la función rgba que vimos antes:

color: hsla(0, 100%, 50%, 0.5);


La cuarta manera de definir un color en css es la codificación hexadecimal. Consiste en escribir un color rgb en notación hexadecimal, es decir, utilizando un carácter numeral seguido de 3 pares de dígitos, cada uno de los cuales representa un valor de 0 a 255 escrito en numeración hexadecimal. El primer par de dígitos corresponde al canal red, el segundo a green y el tercero a blue.

La declaración color:red en codificación hexadecimal luce así:

color:#FF0000;

Pero cuando todas las letras correspondientes a los canales se repiten, como en el ejemplo anterior, donde la F del canal rojo aparece 2 veces y el 0 del canal verde y el 0 del azul también, podemos escribir un único dígito para cada canal:

color:#F00;

Dicha función requiere como parámetro la ruta correspondiente a un archivo de imagen. En el ejemplo usamos por comodidad una imagen externa (alojada en otro dominio), pero siempre conviene trabajar con imágenes locales, es decir, localizadas en el mismo equipo en que se encuentran nuestros archivos html. La imagen de la que hablamos mide 100 pixeles de ancho por 100 pixeles de alto. (100 pixeles [o px] equivalen aproximadamente 2,5 cm.). Evidentemente esa medida es mucho menor que las dimensiones de la pantalla y, como css pretende por defecto cubrir la totalidad del fondo con el valor que asignemos a la propiedad background, lo que hace cuando una imagen es pequeña es repetirla en mosaico hasta cubrir toda la superficie disponible.

    margin:30px;

} 
#tres{

En la primera caja repetimos el background del ejemplo anterior, por lo que la imagen se comporta de manera similar: se multiplica en mosaico hasta rellenar toda su superficie.

A la segunda, luego del último paréntesis correspondiente a la función url, le añadimos un espacio en blanco y el modificador repeat-x. Lo que ese agregado ocasiona es una nueva repetición en mosaico, pero solo en sentido horizontal (coincidente con la dirección del eje x de las coordenadas cartesianas).


A la tercera caja, luego de la función url y de un espacio en blanco, le sumamos el modificador repeat-y, que causa el mismo efecto que el anterior, pero en sentido vertical.

Finalmente, para la última caja reservamos el modificador no-repeat, que despliega la imagen una única vez.

background-size: es una propiedad css que permite redimensionar las imágenes de fondo. Puede asumir como valor las palabras clave contain o cover. O valores en pixeles o porcentaje .

La clave contain hace que la imagen crezca hasta que su ancho o su alto ocupen la totalidad del alto o ancho de la caja. Puede ocurrir entonces que una de las 2 dimensiones de la caja no quede totalmente cubierta por la imagen.

La clave cover hace que la imagen crezca hasta cubrir la totalidad de la superficie de la caja, lo que implica que en algunos casos aparecerá recortada.

Cuando en lugar de claves usamos valores, podemos establecer un único valor o dos. Si establecemos dos, el primero modificará la dimensión X y el segundo la dimensión Y. Si sólo definimos uno, la dimensión X lo adoptará y el correspondiente a la dimensión Y se establecerá automáticamente de manera proporcional. Ya dijimos que X e Y pueden definirse con medidas numéricas fijas o relativas (px o %), pero también pueden asumir el valor auto. Si ambas dimensiones de establecen como auto, la imagen mantiene su tamaño inicial. Si solo uno de los valores se fija a auto, el restante se autodefine con un valor que respeta la proporcionalidad de la imagen. Si definimos para X e Y valores no proporcionales, la imagen se verá deformada.

width, height: son propiedades que se emplean para establecer el ancho y el alto de elementos de bloque e imágenes (no tienen efecto en elementos de línea). Pueden asumir valores fijos o relativos. Si no le asignamos valor a width o le asignamos el valor auto, que es su valor por defecto, el elemento adquirirá el ancho del nodo en que está contenido. En cambio, ante valores de height no definidos o definidos como auto, el elemento tendrá una altura acorde a su contenido.

Hasta ahora, siempre que trabajamos con valores fijos, mencionamos la unidad px (pixel). Pero css admite otras:

  • cm (centímetro)

  • mm (milímetro)

  • Q (cuarto de milimetro)

  • in (pulgada = 2,54 cm)

  • pc (pica o cícero = 16 px)

  • pt (punto = 1,33 px)

Para valores relativos existen otras unidades de medida admisibles, además de porcentaje. Pueden investigarlas utililizando los enlaces que suministramos en el apartado “Para saber más”.

Cuando definimos una medida en porcentaje para asignarla como valor de la propiedad width de un elemento, la misma se establece siempre tomando el ancho de su nodo padre como valor de referencia.

Además, si el elemento cuyo height definimos en porcentaje es hijo directo del tag body, su altura su altura será calculada con referencia al alto de la ventana de visualización de la pantalla, llamado viewport.

padding: es una propiedad css que permite incluir un espacio entre el contenido de un elemento html y su borde.


Cuando hablamos de contenido, como el que se ve en la figura, nos referimos a nodos hijos, es decir, otros elementos html o simplemente texto.

El margen que aparece en la ilustración y del que hablamos anteriormente de manera tangencial, corresponde al espacio de separación que podemos añadir entre el borde de un elemento y el del que lo contiene, o entre el borde de un elemento y el de sus vecinos. Ya lo veremos más detenidamente.

Por defecto, cuando establecemos propiedades de ancho y alto estamos aplicándolas solo al rectángulo marcado en la figura con la palabra contenido. El borde, como quizá ya hayan advertido, agrega espacio al ancho y alto definidos con width y height respectivamente.

Si vuelven a ejecutar el último archivo con que ejemplificamos la propiedad border, van a notar que el cuadrado gris ubicado en el centro del bloque 2 mide los 100 px de lado que le adjudicamos mediante las propiedades width y height. Pero como posee 4 bordes definidos con 50 px de grosor cada uno, el conjunto termina ocupando un espacio de 200 x 200 px.

De la misma manera en que el borde “hace crecer” la caja, veremos que lo mismo sucede con el padding.

La sintaxis de padding tiene la siguiente forma abreviada (shorthand):

padding:10px;

Aplica 10 px de separación entre el contenido y todos los bordes del elemento: superior (top), derecho (right), inferior (bottom) e izquierdo (left).

padding: 10px 20px;

Aplica 10 px de separación para los bordes superior e inferior y 20 px para los bordes derecho e izquierdo.

padding:10px 20px 30px;

Aplica 10 px al borde superior, 20 px a los bordes izquierdo y derecho y 30 px al borde inferior.

padding:10px 20px 30px 40px;

Aplica 10 px al borde superior, 20 al derecho, 30 al inferior y 40 al derecho (es un recorrido en el mismo sentido en que avanzan las agujas de un reloj).

    Pero también es posible definir la separación de cada borde de manera individual:

padding-top:10px;

Aplica 10 px de separación entre el contenido y el borde superior.

padding-right:10px;

Aplica 10 px de separación entre el contenido y el borde derecho.

padding-bottom:10px;

Aplica 10 px de separación entre el contenido y el borde inferior.

padding-left:10px;

Aplica 10 px de separación entre el contenido y el borde izquierdo.

Además de unidades de longitud fijas, la propiedad admite porcentajes. Pero estos se calculan para los 4 lados en referencia al ancho del contenedor.

Si ningún valor es definido, padding asume automáticamente el valor cero.

Asimismo, dentro de los span hemos añadido texto para hacerlos visibles, ya que, como dijimos, width y height son propiedades que no aplican a elementos de línea, los cuales solo crecen en función de su contenido.


En el último bloque hicimos uso de la clave auto, que es, además de las unidades de medida, otro valor que puede asignársele a margin. El valor auto siempre equivale a 0 para top y bottom. Para left y right, si el elemento es de bloque, asume como valor la mitad del espacio horizontal que queda al incluir el elemento. Es por ello que el cuadrado del segundo contenedor (#uno) aparece centrado horizontalmente.

Para elementos de línea auto equivale a cero. Debido a eso, el último span del ejemplo aparece ubicado a la izquierda y separado de #uno solo por el margen de 5 px asignado a ese elemento. El margen de 10 px asignado a #dos no tiene efecto, ya que se trata de un elemento de línea.

font-weight: es la propiedad que permite establecer el grosor del texto. Muchas fuentes admiten solo los valores bold y normal, pero otras admiten una gradación entre esos 2 valores. Los grados, llamados pesos, son los siguientes:

line-height: es la propiedad css con que se especifica el alto de cada renglón de texto. En elementos de línea determina la altura de la caja. Pseudoclases


Las pseudoclases son un tipo especial de selectores. Se utilizan para definir el aspecto que debe asumir un elemento html cuando se encuentra en un estado determinado. Por ejemplo, permite modificar el aspecto de un campo de formulario cuando está enfocado, o el de un checkbox cuando está seleccionado, o el de cualquier elemento cuando tenemos el ratón sobre él.

focus: es la pseudoclase que se pone en marcha cuando un elemento, generalmente un control de formulario o de contenido editable, recibe el foco al ser tocado, activado con un clic o señalado con la tecla tab.

<!DOCTYPE html> 
<html> 
<head> 
<meta charset="utf-8"> 
<title>CSS</title> 
<style>

    width:300px; 
    height:40px; 
    line-height:40px; 
    padding:0 20px; 
    font-family: Arial, Helvetica, sans-serif; 
    font-size:16px; 
    color:#000; 
    outline:none; 
    border:1px solid #000; 
} 
textarea{ 
    height:100px; 
    display:block; 
    margin-top:30px; 
} 
:focus{ 
    background:pink; 
} 
</style> 
</head> 
<body> 
    <input type="text"> 
    <textarea></textarea> 
</body> 
</html>
